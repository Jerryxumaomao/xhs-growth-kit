# CLAUDE.md 模板 —— 拷到你项目根目录，让每个会话的模型都守规矩

> 用法：`cp docs/CLAUDE.template.md <你的项目>/CLAUDE.md`，然后把 `【】` 占位符填成你的实际情况。
> CLAUDE.md 会被 Claude Code **自动注入每个新会话**——这是给未来模型（尤其是较弱模型）修护栏的最高杠杆位置。
> 原则：**弱模型照做就不出错，强模型读"为什么"就能举一反三。**

# 本项目操作守则（每个会话先读这个，再动手）

本项目 = 小红书账号运营台账 + cheat-on-content 校准流水线。**草率操作会污染实验数据或毁掉盲评诚信**。不确定时先读对应 SKILL.md，不许凭感觉执行。

## 🧠 系统原理（强模型读这段，铁律都能自己推出来）

这套系统是给内容创作装**校准回路**：盲评预测 → T+N 实绩复盘 → 偏差 → 进化打分规则。借的是预测市场/科学实验的认识论：

- **预测的价值只在"写于结果之前"**。见过数据再"预测"= 事后归因，偏差信号变噪声。
- **打分者必须与历史数据隔离**。见过实绩的模型打分会不自觉向已知结果回归——所以打分交给只看得到稿子+规则的隔离 sub-agent（channel B），主对话（channel A，天然污染）只做对照。
- **预测不可修改**。允许改 = 允许把预测"修准" = p-hacking，校准池报废。
- **样本量决定动作幅度**。5 样本才第一次 bump，20+ 才数据驱动调权重——防止在噪声上过拟合。

推论：任何"让数字好看一点""这次特殊通融"的请求都在腐蚀回路。**宁可这一篇没有贡献，不能让池子失真。**

## 🚫 五条铁律（附"为什么"）

1. **预测不可变**：`predictions/*.md` 的 `## 预测*` 段写完即锁死（hook 拦截），修笔误也不行；复盘只能追加；重做走 `_redo.md`。
   *为什么：连笔误都不放行，是因为"修笔误"是最常见的篡改借口，一刀切才守得住。*
2. **打分必须盲评**：spawn 隔离 sub-agent（`skills/cheat-score-blind/SKILL.md`），它只许读 script + rubric_notes.md；主对话自估只做 disagreement 对照，绝不落盘。**盲评 agent 拒评时它永远是对的：改输入，不许绕。**
   *为什么：disagreement 表就是主对话污染程度的测量仪（实测主模型会在互动类维度系统性偏乐观）。*
3. **实绩数字永不进 rubric_notes.md**（盲评白名单）；真实标题+数字写 `rubric-memo.md`；`scripts/` 里也不许残留实绩数字。
   *为什么：白名单是盲评 agent 唯一的眼睛，藏一个实绩数字盲就破了。判断标准：数字属于"规则"（bucket 边界/公式）可留，属于"结果"（某篇的播放/涨粉）走 memo。*
4. **state 字段有唯一写者**：`.cheat-state.json` 契约见 `shared-references/state-management.md`；`pending_retros` 是对象数组 `{file, published_at, platform, retro_done, note}`，`retro_done` 非空 = 已复盘。
   *为什么：state 是所有子流程共享的真相源，两个写者 = 语义分叉，下游 hook 会静默失效。*
5. **绝不自动发布**；用户手动发布后说「已发布」再登记。
   *为什么：自动发布有平台封号风险，且发布时机是用户的运营决策。*

## 📁 文件地图（谁可写 / 谁禁写）

| 文件 | 规则 |
|---|---|
| `predictions/*.md` | 预测段锁死；复盘段只追加；header 元数据可补 |
| `rubric_notes.md` | 盲评白名单——通用规则 only，禁实绩/具名数字 |
| `rubric-memo.md` / `audience.md` | 实绩档案，盲评 agent 硬禁读 |
| `scripts/*.md` | 用户定稿前可改；「启动预测」后锁定（再改走 v2） |
| `videos/*/report*.md` | 复盘原始数据；重抓前先备份，不许覆盖旧窗口 |
| 一切改动 | git commit——台账的 git history 就是档案 |

## ⚠️ 高频翻车点（发生一次就写进来一条）

| 你想做 | 必须先 | 为什么 |
|---|---|---|
| 复盘还没到 T+N 整点 | 等整点或标 early_retro | 不同窗口的数据不可比，早采样会被降权 |
| 重抓数据 | 备份原 report.md | 每个时间窗的快照都是不可再生的 |
| 给成片/已发布内容打分 | 做成干净草稿放 scripts/ 再盲评 | 带"成片/已拍"字样的输入会被盲评 agent 正确拒评 |
| 剪辑段落收口 | 查最后一个实字的真实结束时间 | ASR 句尾标点 token 会吞 1-2s 拖尾静音 |
| 改第三方工具源码 | 立刻在其仓库 git commit | 防下次 pull 静默冲掉 |
